The Problem

Why not just use what we have?

Looker / Looker Studio

  • Read-only — can't drive action
  • No APIs, workflows, or AI
  • Filters and charts, not tools
  • Tied to warehouse data; can't pull live CRM state

Admin Tools in Product

  • Competes with product work for engineering time
  • Coupled to the product release cycle
  • Adds surface area to the core product
  • Non-eng teams can't iterate without filing tickets

HubSpot

  • Shows raw activity, not synthesized insights
  • AI limited to HubSpot data only
  • Can't combine with Scoir or other sources
  • Generic CRM views, not purpose-built workflows

Claude Cowork

  • One person, one machine — results aren't shared
  • No persistent UI — every task starts from scratch
  • No shared connectors — each person configures their own
  • Great for ad hoc tasks, not team-wide tools

Productivity Hub

  • AI-native — Claude summarizes and suggests
  • Any data source — HubSpot, Scoir, Slack in real time
  • Custom logic — scoring, classification, workflows
  • Decoupled — ships independently from the product
  • Shared — one tool the whole team uses

Applet channels

Ship experiments without breaking what works. Every applet supports two channels — :latest (stable) and :next (experimental). Because both channels run in production side-by-side, there’s no need for a separate feature environment — :next is the preview, running against real data.

Enhance
/enhance-applet
Fork into :next
→
Test
/deal-planner/next
Team plays with it
→
Promote
/promote-applet
:next becomes :latest

/enhance-applet

Create a :next channel for any applet

/enhance-applet "deal-planner"

# Claude will:
# 1. Copy the applet into a next/ channel
# 2. Ask what you want to change
# 3. Build the enhanced version
# 4. Deploy alongside :latest

How it works

app/sales/deal-planner/
  page.tsx              # :latest (unchanged)
  dashboard.tsx
  next/                  # :next channel
    page.tsx            # enhanced version
    dashboard.tsx
  • Convention-based — a next/ subdirectory is all it takes
  • Zero config — Next.js routes it automatically
  • Sidebar badge shows which applets have a :next channel
  • Promote or discard — no leftover state
  • No feature environment needed — test changes in production without risk, since :latest is untouched

Shared infrastructure

Connectors

Singleton API clients in lib/connectors/ — shared across all applets. No per-applet config needed.

HubSpot

lib/connectors/hubspot/

  • Deals, contacts, companies, owners, pipelines
  • Email history via associations API
  • Search with filters (owner, pipeline, stage)
  • Record URL builder

Anthropic (Claude AI)

  • ask() — simple text-in / text-out

Server-first data flow

How auth works today

All Scoir apps — admissions-web, student-web, counselor-web, and others — share the same single sign-on system. Log in once, and every app recognizes you. Each app checks your permissions to control what you can see and do.

How Productivity Hub fits in

  • Same login — no separate account or credentials needed
  • Permission-gated applets — sidebar only shows what you have access to
  • Scoir data access — authenticated requests unlock product data alongside HubSpot and other sources
  • Deep links to Scoir — applets that reference schools or colleges link directly into admissions-web or counselor-web
